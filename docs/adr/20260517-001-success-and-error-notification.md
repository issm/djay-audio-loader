# 成功・エラー通知の実装

## Status

Accepted

## Context

issue #4 で error レベルのログを macOS 通知で表示する仕組みを `logger.rs` に組み込んだが、成功時の通知（どの曲をどのデッキにロードしたか）は未実装だった。ホットキー経由で実行するユースケースではターミナルを見ないため、成功・失敗を問わずフィードバックが必要。

通知に含めたい情報:
- 成功時: デッキ番号、アーティスト名、曲名、コメント（あれば）
- エラー時: エラーメッセージ

## Decision

`src/notify.rs` を新規作成し、成功通知とエラー通知を専用モジュールとして分離する。

- 通知方法: `osascript` の `display notification`（issue #4 と同方式）
- 成功通知: ロード完了後に `main.rs` から呼び出し
- エラー通知: `main.rs` の `run()` でエラー発生時に呼び出し
- サウンドなし（DJ 中に邪魔にならないよう）
- アートワーク画像の添付は別 issue（#10）に分離

## Consequences

- ロード成功時に曲名・デッキ番号が通知で確認でき、操作の確実性が向上
- エラー通知が `logger.rs` 内と `notify.rs` の2箇所に存在する形になった（logger 側は汎用、notify 側は構造化された内容）
- `display notification` の制約で画像添付ができない（→ issue #10）
