# ホットキー監視ヘルパの分離

## Status

Accepted

## Context

`drag-into-djay` は単発実行の CLI ツールとして設計されている。実際の運用ではホットキー一発でデッキにロードしたいが、CLI を都度手動実行するのは非実用的。

方法の候補:

1. **`drag-into-djay` 自体にホットキー監視を組み込む** — 単一バイナリだが責務が混在する
2. **別バイナリとしてヘルパを用意し、ホットキー押下時にサブプロセスとして呼び出す** — 責務が明確に分離される
3. **外部ツール（skhd, Hammerspoon 等）で起動** — 外部依存が増える

## Decision

`djay-audio-loader-helper` を別バイナリとして作成する。CGEventTap でグローバルホットキーを監視し、押下時に `drag-into-djay` をサブプロセスとして起動する。

- ホットキーイベントは消費（他アプリに伝達しない）
- 多重起動防止（実行中の再押下はスキップ）
- 同じディレクトリに両バイナリを配置する前提

## Consequences

- 責務が明確に分離され、`drag-into-djay` は単独でもテスト・利用可能
- 2つのバイナリを管理する必要がある
- ヘルパのオンオフ（起動・停止）の仕組みを別途検討する必要がある（→ issue #2）
- ホットキー設定がハードコードされている（→ issue #3）
