# ドロップゾーン検証ログ

## 目的

djay Pro の波形エリアへのファイルロードを、カーソルを奪わずに実現できるか検証する。

---

## 確定した座標情報

`dump-all-elements.applescript` の実行結果より：

| 要素 | デッキ | position | size | 中央座標 |
|---|---|---|---|---|
| 波形 | デッキ1 | 613, 194 | 694 x 119 | (960, 253) |
| 波形 | デッキ2 | 613, 314 | 694 x 119 | (960, 373) |

- 両デッキの波形エリアは水平方向の範囲が同じ（x: 613〜1307）
- 垂直方向に上下に並んでいる

---

## 検証フロー

### Step 1: AXUIElement による値セット

スクリプト: `.kiro/scripts/verify-axuielement.applescript`

```sh
osascript .kiro/scripts/verify-axuielement.applescript 2>&1
```

波形要素に対して `set value` でファイルパスをセットできるか確認。
対応していれば、カーソルを奪わずにロード可能。

**結果**: ❌ 失敗。`set value` はエラーなく成功するが、djay Pro 側では無視されロードされない。

---

### Step 2: CGEvent による PID 指定ドラッグ&ドロップ

スクリプト: `.kiro/scripts/verify-cgevent-drag.swift`

```sh
swift .kiro/scripts/verify-cgevent-drag.swift /path/to/test.mp3 960 253  # デッキ1
swift .kiro/scripts/verify-cgevent-drag.swift /path/to/test.mp3 960 373  # デッキ2
```

`CGEvent.postToPid()` で特定プロセスへイベントを送信し、カーソルを奪わずにドラッグ&ドロップできるか確認。

**結果**: ❌ 失敗。`postToPid()` でマウスイベントは送れるが、ファイルのドラッグ情報（NSDraggingSession）が伝わらずロードされない。

---

### Step 3: cliclick によるドラッグ&ドロップ（フォールバック）

上記2つが失敗した場合のフォールバック。カーソルを奪うが、使用シーンはフォアグラウンド前提のため許容。

```sh
brew install cliclick
cliclick dd:/path/to/test.mp3:960,253  # デッキ1
```

**結果**: ❌ 却下。`cliclick` はファイルを伴うドラッグ&ドロップに非対応（座標間のドラッグのみ）。

---

## 方針転換

カーソルを奪わない方法はすべて失敗・却下となったため、アプローチを根本的に変更する。

### 新方針

ドラッグ元（Swinsian の選択中アイテム）とドラッグ先（djay Pro の波形エリア）の両方が実在する UI 要素であることを前提に、**CGEvent で実際のドラッグ&ドロップをシミュレート**する。

- ドラッグ元: Swinsian の選択中アイテムの座標を Accessibility API で動的取得
- ドラッグ先: djay Pro の指定デッキの波形エリア座標を Accessibility API で動的取得
- カーソルは奪われるが、使用シーンはフォアグラウンド前提のため許容

### ツール構成

詳細は `.kiro/specs/requirements.md` を参照。

- `drag-into-djay`: ドラッグ&ドロップ実行ツール（Rust）
- `djay-audio-loader-helper`: ホットキー監視デーモン（djay Pro 起動をトリガに監視開始・終了）
